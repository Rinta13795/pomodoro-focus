# Claude Code Prompt 万能模板

## 模板一：修 Bug

```
Bug：[用一句话描述现象，越具体越好]

复现步骤：
1. [第一步操作]
2. [第二步操作]
3. [看到什么错误/异常]

期望行为：[应该发生什么]

修复要求：
1. [具体要改什么]
2. [具体要改什么]

检查点：[可能的原因提示，帮 Claude Code 缩小排查范围]
```

### 实际例子：
```
Bug：点击"开始专注"后取消密码输入，按钮卡在"启动中..."状态，无法操作。

复现步骤：
1. 打开应用，点击"开始专注"
2. 弹出管理员密码框，点击"取消"
3. 按钮显示"启动中..."且不可点击

期望行为：取消密码后，按钮恢复为"开始专注"，可以重新点击。

修复要求：
1. 按钮文字从"启动中..."恢复为"开始专注"
2. 移除 disabled 状态
3. 保持"未开始"状态

检查点：可能只处理了密码验证成功的路径，没有处理失败/取消路径。
```

## 模板二：新功能

```
功能：[一句话说明要做什么]

背景：[为什么要做这个，可选]

实现要求：
1. [具体要求]
2. [具体要求]
3. [具体要求]

视觉/交互：[长什么样，怎么操作，可选]

技术提示：[你知道的技术细节，帮 Claude Code 少走弯路，可选]
```

### 实际例子：
```
功能：专注结束时播放提醒音效

实现要求：
1. 倒计时到 00:00:00 时自动播放提示音
2. 用 Web Audio API 生成（880Hz 正弦波，0.5 秒 × 3 次，间隔 0.3 秒）
3. 设置页添加开关"完成提醒音"，默认开启
4. config.json 新增字段 play_completion_sound: true
```

## 模板三：改进/优化

```
当前问题：[现在是什么样，哪里不好]

改成：[想要什么样]

参考：[有参考的话写上，比如"参考 macOS 时钟 app"]

具体要求：
1. [要求]
2. [要求]
```

### 实际例子：
```
当前问题：时间选择用 [- 25分钟 +] 按钮，操作不方便

改成：参考 macOS 时钟 app，直接点击数字位输入时间

具体要求：
1. 显示格式 HH:MM:SS，点击某一位高亮选中
2. 键盘数字键直接输入，自动跳下一位
3. 左右箭头键在各位之间移动
4. 小时 0-3，分钟 0-59，秒 0-59，最小 1 分钟
```

---

## 写 Prompt 的核心原则

1. **先说现象，再说期望** — 不要只说"有 bug"，要说"我做了 X，出现了 Y，应该是 Z"
2. **一个 prompt 一个问题** — 不要一次塞 5 个不相关的修改
3. **给具体数值** — 不说"时间不能太长"，说"小时最大 3"
4. **不用教它怎么写代码** — 说清楚"要什么"就行，不用写伪代码
